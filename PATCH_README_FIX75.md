# FIX75 — frontend data orchestration / transport pressure reduction

## Что исправлено

- Снижено количество одновременных запросов на Dashboard:
  - ManualOrderPanel теперь использует shared query keys для `/state` и `/settings`
  - список ордеров ручной панели не грузится, пока панель свернута
  - adaptive runtime overview в InstrumentSelector больше не запрашивается постоянно, а только при открытом селекторе
  - ChartContainer перестал дублировать первичный fetch свечей через `useQuery + refetch()` на mount
  - ChartContainer использует общий positions query

- Снижена нагрузка на AccountPage:
  - тяжёлые validation / performance / ML панели больше не грузятся автоматически при первом открытии страницы
  - для них добавен явный переключатель `Загрузить heavy-панели`

- Смягчена глобальная error-сигнализация фронта:
  - optional analytics endpoints больше не объявляют весь backend "умершим"
  - ошибки тяжёлой аналитики остаются локальными для страницы

- Улучшено поведение графика:
  - history fetch идёт контролируемо
  - при пустом ответе или ошибке показывается локальное сообщение вместо бесконечного визуального зависания

## Ожидаемый эффект

- меньше конкурирующих запросов на старте
- меньше ложных красных banner/toast ошибок
- Dashboard и Account должны открываться заметно стабильнее
- страницы перестают выглядеть как "всё сломано", если не отвечает только тяжёлая аналитика
