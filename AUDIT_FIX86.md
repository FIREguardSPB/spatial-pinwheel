# Audit FIX86

## Что подтверждено

### Backend
- Auto-policy runtime path больше не обязан каждый раз заново пересчитывать тяжёлые бизнес-метрики в coordinator read-path.
- Coordinator endpoints теперь получают безопасный auto-policy payload вместо потенциального зависания.
- Candle API больше не подсовывает пользователю синтетические свечи.
- Signal feed summary теперь ближе к фактической свежести сигналов, а не только к времени свечи.

### Frontend
- Приложение теперь реально поднимает backend runtime bootstrap и SSE-подписку при старте.
- SSE-инвалидация затрагивает coordinator pages, а не только legacy query keys.
- Persist старых свечей убран, что должно снизить риск «замороженного» графика после reload.
- График теперь нормализует timestamps и дедуплицирует свечи, что адресует симптом «две ветки свечей».

## Остаточные риски

1. Если worker или market stream реально не публикуют новые kline events, UI теперь будет честно показывать отсутствие обновления, а не маскировать это mock-данными.
2. Если T-Bank/API недоступны и candle cache пуст, график покажет реальную ошибку загрузки свечей — это ожидаемое поведение после запрета mock fallback.
3. Полный runtime smoke-test с вашим live окружением (API + worker + DB + broker) в этой среде не поднимался, поэтому сетевые/инфраструктурные эффекты после деплоя нужно проверить отдельно.

## На что смотреть после развёртывания

1. Dashboard:
   - одна линия свечей;
   - время последней свечи двигается;
   - кнопка обновления реально перетягивает новые данные.
2. Settings:
   - auto-policy / ML / protective contour перестают быть пустыми;
   - runtime overview по бумаге открывается без ошибок.
3. Signals:
   - последний сигнал совпадает с реальным временем создания новых сигналов;
   - в таблице различаются `created_ts` и `signal.ts`.
4. Network:
   - есть стабильный `/api/v1/stream` (SSE);
   - coordinator endpoints `/api/v1/ui/...` не застревают на таймаутах.
