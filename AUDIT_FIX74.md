# AUDIT FIX74

## Главный фактический корень фронтовой поломки
По скриншотам и network trace видно, что фронт ходил напрямую в `http://127.0.0.1:8001/api/v1/...`, а не через относительный `/api/v1`.
Это приводило к `ERR_CONNECTION_REFUSED`, pending XHR, пустым панелям, ложным degraded/error banner и нерабочему stream.

## Что изменено
Исправлен транспортный слой фронта: единый runtime resolver для REST/SSE/export.

## Что это должно исправить визуально
- dashboard должен начать грузиться без лавины pending запросов;
- stream/status перестанут показывать массовую деградацию только из-за неверного host;
- страницы `Signals`, `Trades`, `Settings`, `Dashboard` должны начать получать реальные ответы вместо подвисания.

## Что ещё может остаться после FIX74
Если backend реально не поднят или proxy не настроен, UI всё ещё покажет деградацию — но уже по настоящей причине, а не из-за фронтового hardcoded host.
