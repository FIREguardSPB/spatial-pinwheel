# FIX35

## Что исправлено

1. **Убран наиболее вероятный источник полного моргания/white screen в dev-режиме**:
   - Vite больше не следит за `backend/**`, `*.log`, `*.db`, `__pycache__`, `*.pyc` и похожими backend/runtime файлами.
   - Это должно убрать full-page reload, если фронтенд перезагружался из-за изменений backend/log/db файлов внутри общего репозитория.

2. **SSE больше не гоняется через Vite proxy в dev**, если задан `VITE_DEV_PROXY_TARGET`:
   - `EventSource` теперь подключается напрямую к backend.
   - Это снижает шанс циклических разрывов из-за dev-proxy.

3. **Обычные API-запросы в dev тоже идут напрямую в backend**, если задан `VITE_DEV_PROXY_TARGET`.

4. **Смягчена логика reconnection на фронте**:
   - краткий `onerror` не переводит UI сразу в аварийное состояние;
   - reconnect считается проблемой только после устаревшего heartbeat;
   - жёсткий disconnect — только после более длинного таймаута.

5. **Убран React.StrictMode** из dev-root, чтобы исключить лишние двойные mount/unmount-циклы и дублирующие подключения.

## Важно

После этого патча нужно **полностью перезапустить frontend dev server**.
Просто обновить страницу недостаточно, потому что изменения в `vite.config.ts` применяются только после полного рестарта Vite.
